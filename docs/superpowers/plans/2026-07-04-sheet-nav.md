# Sheet Navigation Strip Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a nav strip to each sheet's dashboard linking back to the landing page and across to the other sheet(s), per `docs/superpowers/specs/2026-07-04-sheet-nav-design.md`.

**Architecture:** `renderIndex` gains an optional `sheetNav` 4th param (default `[]` = render nothing, backwards compatible); `run-report.js` builds the nav list once from its existing `groups` map and passes it per sheet with a `current` flag. Display layer only — `node src/run-report.js` applies it.

**Tech Stack:** Node ESM, `node:test` via `npm test` ONLY (`node --test test/` breaks on newer Node).

## Global Constraints

- Report UI Thai via `src/report/labels.js` (new strings go in `T`); contract values + CSS class names English.
- `renderIndex(rows, systemicCount, chromeCount = 0, sheetNav = [])` — default `[]` renders NO strip; existing callers/tests must pass unchanged.
- Strip appears ONLY on sheet dashboards; detail/systemic/chrome/criteria pages untouched.
- Current sheet renders as `<b>` text, never a link; landing link is `../index.html`; other sheets `../<slug>/index.html`.
- Tests via `npm test` only; suite stays green. NEVER run run-capture/run-compare (display change → run-report only).

## File Structure

### Task 1: `renderIndex` sheet nav strip

**Files:**
- Modify: `src/report/labels.js` (add one `T` key)
- Modify: `src/report/html.js` (`renderIndex` signature + markup)
- Test: `test/html.test.js` (append)

**Interfaces:**
- Consumes: `esc`, `T` (existing).
- Produces: `renderIndex(rows, systemicCount, chromeCount = 0, sheetNav = [])` where `sheetNav: {name: string, slug: string, current: boolean}[]` — Task 2 calls it with this exact shape.

- [ ] **Step 1: Write the failing tests**

Append to `test/html.test.js`:

```js
test('renderIndex renders sheet nav strip: landing link, other-sheet link, current as bold', () => {
  const nav = [
    { name: 'TH Pages - Categorized', slug: 'th-pages-categorized', current: true },
    { name: 'News & Media Articles', slug: 'news-and-media-articles', current: false },
  ];
  const html = renderIndex([], 0, 0, nav);
  assert.match(html, /<a href="\.\.\/index\.html">← หน้ารวม<\/a>/);
  assert.match(html, /<a href="\.\.\/news-and-media-articles\/index\.html">News &amp; Media Articles<\/a>/);
  assert.match(html, /<b>TH Pages - Categorized<\/b>/);
  assert.doesNotMatch(html, /<a href="\.\.\/th-pages-categorized\/index\.html"/);
});

test('renderIndex without sheetNav renders no nav strip (backwards compatible)', () => {
  const html = renderIndex([], 0, 0);
  assert.doesNotMatch(html, /หน้ารวม/);
  assert.doesNotMatch(html, /class="toplinks sheetnav"/);
});
```

- [ ] **Step 2: Run tests to verify they fail**

Run: `npm test`
Expected: FAIL — first new test (no strip markup exists yet); second may pass already (assert it anyway as the regression lock).

- [ ] **Step 3: Implement**

In `src/report/labels.js`, add to `T`:

```js

```

In `src/report/html.js`, change the `renderIndex` signature:

```js
export function renderIndex(rows, systemicCount, chromeCount = 0, sheetNav = []) {
```

Add above the `return` statement (next to the other consts):

```js
  const sheetNavLine = sheetNav.length > 0
    ? `<p class="toplinks sheetnav"><a href="../index.html">${T.backToLanding}</a> · ${sheetNav
        .map((s) => s.current
          ? `<b>${esc(s.name)}</b>`
          : `<a href="../${esc(s.slug)}/index.html">${esc(s.name)}</a>`)
        .join(' · ')}</p>`
    : '';
```

Insert `${sheetNavLine}` in the returned template directly after the `<h1>` line, above the existing toplinks paragraph:

```js
<h1>${T.reportTitle}</h1>
${sheetNavLine}
<p class="toplinks"><a href="criteria.html">เกณฑ์การตรวจสอบ</a>${systemicLink}</p>
```

(The existing toplinks line already carries `${systemicLink}` — leave it exactly as-is; only the `${sheetNavLine}` line is new.)
